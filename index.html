Part 1: MCP Setup Walkthrough

MCP (Model Context Protocol) is what separates casual Cowork users from power users. It connects Claude to your actual tools — Slack, Notion, Gmail, Calendar — so it can read and act on real data, not just respond to prompts. Here's the exact setup, step by step.

1

Download Claude Desktop and Enable Cowork

Go to claude.ai/download. Install the desktop app. In Settings, toggle on Cowork (beta). This unlocks computer use and multi-step agentic runs — without this, you're just using a chat interface.

2

Connect Gmail via MCP

In Cowork Settings, go to Integrations and select Gmail. Click Connect and authorize via OAuth. Once connected, Cowork can read your inbox, search threads, and draft replies. Test it: type "what are my 3 most recent emails?" and it should respond with real data.

3

Connect Slack via MCP

Same flow — Integrations, then Slack, then OAuth. Grant channel read access. Cowork can now search messages, pull context from specific channels, and read threads. Test: "what was discussed in #general yesterday?" — if it returns real messages, you're connected.

4

Connect Notion via MCP

In Integrations, connect Notion. You'll need to authorize at the workspace level and select which pages Cowork can access. Give it access to your main workspace pages — it needs to be able to read and write to be useful. Test: "find my latest project notes in Notion."

5

Connect Google Calendar via MCP

Connect Calendar the same way. Once live, Cowork can see your schedule, pull upcoming meetings, and use your availability as context for any task. Test: "what does my week look like?" — if it shows your real calendar, you're done.

6

Verify Everything is Working

Run this single prompt to confirm all 4 connections are live:

give me a quick status check: what's in my inbox right now, what's on my calendar today, do i have any unread slack messages, and find my most recently edited notion page.

If it returns real data from all 4 sources, your MCP stack is fully operational.

Part 3: The Agent Stack Architecture

The biggest unlock from the course: Cowork is not a single AI assistant — it's a coordination layer. Once you understand agent stacking, you go from 1x output to 5x output without doing any more work yourself.

The core insight: instead of one big prompt that tries to do everything, you chain smaller agents that each do one thing well — and pass their output to the next agent. Think of it like an assembly line, not a single worker.

1

The Trigger Agent (Input Layer)

The first agent in any stack is a "trigger" — it gathers raw data from your tools (Gmail, Slack, Notion, Calendar) and outputs a structured summary. It does nothing else. This agent is always simple: "pull X from Y and format it as Z."

trigger agent example: pull all new inbound leads from gmail and notion from the last 7 days. output as a clean JSON list: name, company, email, stated problem, date received. do nothing else. do not score or respond — just collect and format.
2

The Processing Agent (Middle Layer)

The second agent takes the trigger's output and does the actual thinking — scoring, categorizing, analyzing, or writing. It receives structured input and outputs structured results. Never give it raw, unformatted data.

processing agent example: here is my lead list: [paste trigger agent output] score each lead 1-10 based on: company size, title seniority, problem fit, budget signals. output: same list with a score, a 1-sentence rationale, and a priority tier (hot/warm/cold).
3

The Output Agent (Action Layer)

The final agent takes the processed results and creates deliverables — emails, documents, Notion updates, or Slack messages. It only receives clean, scored data. This is the agent that "does" things in the world.

output agent example: here are my hot leads: [paste scored list, hot tier only] for each one, draft a personalized outreach email. reference their specific company and problem. no templates. no "I hope this finds you well." output: one draft per lead, ready to send.
4

Running Agents in Parallel

Once you've built individual agent chains, you can run multiple stacks at the same time in separate Cowork windows. While your lead scoring stack runs in window 1, your content stack runs in window 2, and your inbox triage runs in window 3. You're not waiting — you're reviewing outputs while the next batch processes.

Part 4: Advanced Prompt Patterns

These are the prompt engineering patterns specific to Cowork that the course spent the most time on. Generic ChatGPT advice doesn't apply here — Cowork has computer use and tool access, which changes how you should prompt it.

Pattern 1: Context Injection
Always tell Cowork who you are and what your business does at the start of complex tasks. It doesn't retain memory between sessions by default. Without this, it gives generic answers.
context: i'm [Name], [role] at [Company]. we sell [product] to [ICP]. my communication style is [direct/formal/casual]. my top priority this week is [X]. with that context, [your actual task here].
Pattern 2: Output Format Specification
Cowork defaults to long, paragraphed responses. If you want scannable output — bullets, tables, or specific formats — you have to specify it explicitly every time.
format your output as: - a 3-bullet executive summary at the top - then a table with columns: [column 1] | [column 2] | [column 3] - then any action items as a numbered list - no paragraphs. no preamble. start directly with the summary.
Pattern 3: The Constraint Frame
Cowork's biggest failure mode is doing too much when you want it to do one specific thing. Constraints prevent scope creep and keep outputs focused.
constraints for this task: - only use data from the last 7 days - do not make assumptions — if you're unsure, flag it and ask - output maximum 5 items, prioritized by importance - do not write more than 2 sentences per item
Pattern 4: The Voice Lock
When Cowork writes anything on your behalf (emails, posts, updates), give it 3 examples of your actual writing first. This locks in your voice and eliminates the "AI-generated" smell.
here are 3 examples of how i actually write: example 1: [paste a real email or message you wrote] example 2: [paste another] example 3: [paste another] study these. now write [your actual task] in the exact same voice. same tone, same sentence length, same level of formality.
Pattern 5: The File System Trick
Cowork with computer use can read files directly from your machine. Store your most important context (ICP definition, company one-pager, voice guide) as .txt files on your Desktop. Then reference them directly in prompts instead of copy-pasting every time.
read the file at ~/Desktop/company-context.txt and use it as background context for everything in this session. do not summarize it back to me — just use it silently to inform every response.

Part 5: The 20-Minute Setup Checklist

Go from zero to operational in 20 minutes. Do these in order — each step builds on the last.

⬜

Minutes 0-3: Install and enable

Download Claude Desktop, sign in, toggle on Cowork mode in Settings.

⬜

Minutes 3-8: Connect your 4 tools

Gmail, Slack, Notion, Google Calendar — one OAuth each. Takes about 90 seconds per tool.

⬜

Minute 8-9: Run the connection test

Use the verification prompt from Part 1. Confirm all 4 data sources return real data.

⬜

Minutes 9-11: Create your context file

Make a .txt file on your Desktop called "company-context.txt". Add: your name, role, company, ICP, and communication style. You'll reference this in every session.

⬜

Minutes 11-14: Run your first real automation

Pick automation #01 (Morning Brief) from Part 2. Run it. See real output from your actual tools.

⬜

Minutes 14-17: Build your first agent chain

Take any task you do weekly and break it into a 3-agent stack (Trigger / Process / Output) using the framework from Part 3.

⬜

Minutes 17-20: Save your top 3 prompts

Copy the 3 automations from Part 2 that apply most to your work. Paste them into a Notion page called "Cowork Prompts." You'll use these every week.
